🚀 Beyond Static Maps: Building an Interactive Geospatial Dashboard for Odisha’s Agriculture

How do rainfall patterns and fertilizer inputs actually translate to crop yields at the grassroots level? To explore this, I took spatial data for Odisha's districts and built an interactive GIS web application that brings these agricultural variables to life.

Initially, I had separate, static visual outputs for rainfall, rice yield, and fertilizer application. While informative, looking at them separately made it challenging to spot direct correlations.

To elevate the project, I used QGIS and the qgis2web plugin to deploy a dynamic web map using the Leaflet engine.

Key Features of the Dashboard:
🔹 Multi-Layer Toggle: Users can seamlessly switch between Average Rainfall (mm), Rice Yield (Kg/Ha), and Fertilizer Applied (kg/Ha) via an expanded layer menu to visually correlate inputs vs. outputs.
🔹 Data-Driven Symbology: Utilized Natural Breaks (Jenks) grading to accurately cluster agricultural thresholds, ensuring true geographic representation.
🔹 Clean UX/UI: Configured interactive popups utilizing "visible with data" labels to completely eliminate backend database noise, alongside district highlight-on-hover actions.
🔹 Built-in Search: Integrated a localized search layer allowing users to query specific districts (like Cuttack or Ganjam) and zoom instantly to their metrics.

This project highlighted how web-mapping tools can bridge the gap between raw data tables and stakeholders, enabling interactive exploration of critical agricultural insights.
